如何用Hexo與GitHub搭建個人部落格

Feb 23, 2020 | 閱讀

某天天外飛來一筆，想把自己目前散在網路各處的文章段落整理好放在電腦裡，也想順便放上網路，剛好以前學過一咪咪的 Github，記得它有一個 Pages 功能可以用，於是關鍵字 github+blog 直接丟進谷哥搜尋，找到了 Jekyll 這玩意，網頁寫得好吸引人呢，這個那個裝好，就有部落格可以用了喲！所以當天下班回家馬上開工，但對 coding 小菜鳥的我而言，還是太難，整個安裝過程非常不順，灰心又眼睛痛之餘發現了 Hexo，比起 Jekyll 採用的 Ruby，Node.js 打底而生出的 Hexo 不知為何讓我更容易理解，各家資料邊看邊做之下，也順利建置完了自己的部落格。(也可能因為我發現可以用中文關鍵字找中文資料所以容易理解…對，我也不知道我腦袋怎麼回事。)

好的，前言說完了，以下列出我這菜鳥摸索出來的詳細步驟，腦容量很小的我以後可以回來參考。

這時再去 D 槽的 Apps 裡，應該就可以發現剛剛建立的 my_blog，裡面有一些有的沒的資料夾跟檔案，如：

  • _config.yml：網站配置檔案，可以在此配置大部分的設定。
  • package.json：套件版本控制
  • scaffolds：鷹架資料夾。建立新文章時，會根據 scaffold 來建立檔案。
  • source：原始檔案資料夾是放置內容的地方。
  • themes：主題資料夾。Hexo 會根據主題來產生靜態檔案。

更換主題

Hexo安裝完畢後本身附帶一個預設的樣式主題 /themes/landscape，就是剛剛預覽所看到的長相，如果想要看看還有什麼樣的主題，可以到這裡來看看，琳琅滿目供君選擇。

很多人選擇用 NexT，不過反骨的我選擇使用這一個：maupassant。選擇主題的時候最好順便谷歌一下，看看文章數來估計一下使用的人多不多；因為用得人多，才容易查得到各種指導文，如果你硬是選了一個沒啥人用的主題，你之後就只能好自為之ㄌ。我是叛逆的選來裝好後，陸續想到一些想改的樣式跟想加的功能，動手整修的時候才驚覺為什麼大家都用 NexT，不過還好我選擇的主題也還算 popular，所以化險為夷（這成語應該可能並不是這樣用）。
